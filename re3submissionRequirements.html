re3data.org Submission Requirements

Essential criteria and data fields for a new repository.

Minimum Requirements

  • ✓ **Focus on Research Data:** Must be dedicated to research data.
  • ✓ **Legal Entity:** Operated by a sustainable organization (e.g., university).
  • ✓ **Clear Access Conditions:** Access to repository and data must be clearly explained.
  • ✓ **Terms of Use:** Must have clearly stated terms of use.

Key Data Fields

  • ● **Name & URL:** Official name and web address of the repository.
  • ● **General Information:** A short description, content types, and keywords.
  • ● **Responsibilities:** Who is responsible for funding and content.
  • ● **Policies & Legal:** Repository policies, licenses for the data and database.
  • ● **Technical Standards:** APIs, dataset versioning, and software used.
  • ● **Quality Standards:** Any certifications or audit processes.

re3data.org Submission Process

The submission process for a new repository to re3data.org is straightforward. It begins with the submitter providing basic information about the repository. This is followed by a review process where the re3data.org team assesses the repository against the minimum requirements. Once approved, the repository is indexed and becomes discoverable on the platform.
